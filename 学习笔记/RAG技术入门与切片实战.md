# RAG 技术入门与切片实战

## 概述

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将外部知识检索与大模型生成相结合的技术范式。本质是给模型的一场**开卷考**：先检索，再生成。

> **一句话理解**：RAG 就像开卷考试——学生（模型）不需要记住所有知识，只需要会翻书（检索）和理解（生成）就够了。

---

## 为什么需要 RAG？

传统大模型的三大短板：
- **知识过时**：模型训练后无法自动获取新知识
- **幻觉问题**：面对未知问题会"一本正经地胡说"
- **私有数据隔离**：无法访问企业/个人的私有数据

### RAG vs 微调：为什么 RAG 是首选？

| 维度 | RAG | 微调 |
|------|-----|------|
| 成本效益 | 低（无需训练模型） | 高（需要算力和数据） |
| 知识时效性 | 实时更新（换文档即可） | 需要重新训练 |
| 回答可塑性 | 灵活（换知识库即换能力） | 固化在模型中 |
| 落地门槛 | 低 | 高 |

---

## RAG 完整工作流程

### 离线索引阶段

将海量文档转化为结构化的向量知识库：

```
数据加载 → 文本切片 → 向量化 → 向量存储
```

### 在线推理阶段

针对用户提问进行精准检索 + 生成回答：

```
用户提问 → 向量召回 (Retrieve) → Rerank 重排序 → 送入大模型 → 生成回答
```

> 完整固定顺序：**文档切片 → 向量化入库 → 向量召回 → Rerank 重排序 → 送入大模型**

## 文本切片策略

文本切片是 RAG 效果的**核心决定因素**。

### 为什么要做文本切片？

1. **适配模型上下文**：模型上下文窗口有限
2. **提升检索精度**：小片段更精准匹配
3. **减少冗余信息**：过滤无关内容

### 六种切片方式

#### 1. 固定长度字符切片

- **原理**：按预设字符数（如每 1000 字符）均匀切分
- **优势**：实现简单，速度极快
- **局限**：容易在句子中间截断，破坏语义完整性
- **适用**：快速原型验证

#### 2. 固定长度 Token 切片

- **原理**：按固定数量的 Token（如每 512 个）切分
- **优势**：精准控制长度
- **局限**：语义破坏风险
- **适用**：对长度敏感的简单文本

#### 3. 父子文档切片

- **原理**：双重存储与分离机制——小块用于检索，大块用于提供上下文
- **优势**：兼顾精度与完整性
- **局限**：成本与复杂度提升
- **适用**：强上下文依赖的长文本问答

#### 4. 大模型切片

- **原理**：通过精细提示词引导大模型进行语义切分
- **优势**：极致智能
- **局限**：成本与效率瓶颈
- **适用**：极高精度的专业文本

#### 5. 递归字符切片（LangChain）

- **原理**：分层递归——先按段落/章节切分，超限则降级为句子级切分
- **优势**：保留语义完整性
- **局限**：非标准文本场景效果下降
- **适用**：通用场景，**效果与效率的最佳平衡点**

#### 6. 语义切片

- **原理**：利用嵌入模型计算句子间语义相似度，低于阈值时判定"主题漂移"并切分
- **优势**：语义无损
- **局限**：计算成本高
- **适用**：逻辑复杂的长文本、高要求的专业领域

### 切片方式选型总结

| 场景 | 首选方案 | 理由 |
|------|----------|------|
| 快速原型 / 通用场景 | **递归字符切片** | 效果与效率的最佳平衡 |
| 对长度敏感 / 简单文本 | 固定长度 Token 切片 | 精准控制 |
| 高精度需求 / 复杂长文本 | **语义切片** | 语义无损 |
| 长文本问答 / 需要完整上下文 | **父子文档切片** | 双层索引，兼顾召回率和理解力 |
| 超高精度 / 预算充足 | 大模型切片 | 极致智能 |

## 重叠度（Overlap）

切块时前后两块重复的文字，防止把一句话或一段逻辑硬生生切断，**提升 RAG 问答准确率**。

---

## Rerank（重排序）

**Rerank** 只做一件事：输入「用户问题 + 一段文本片段」，输出一个相关性分数，分数越高越匹配，最后按分数重新排序。

流程：先召回，再重排序（Rerank）。

## 常见问题排查

### 知识库回答不符合预期

明确知识库有对应片段时，排查顺序：
1. 看**原文档**有无知识片段
2. 看**切片**后有无这个片段
3. 看**检索结果**有无知识片段（top_k）
4. 看**提示词**是按知识库回答还是忽略知识库回答

### 模型忘记刚才说了什么

- 未超出上下文：加强提示词
- 即将超出：压缩上下文

### 上下文污染

确实想重开对话时，可以**重开一个对话**（配合强提示词）

### 模型部署后逐渐爆内存

原因：模型推理时显存/内存没有被正常释放，越堆越多。
排查：检查是否没有清理缓存；历史对话/向量未释放；模型被重复加载。
