# 实时通信技术

## 概述
HTTP 的请求-响应模型不适合实时场景。本文介绍从简单到复杂的四种实时通信方案：短轮询、SSE、WebSocket，以及技术选型指南。

## 核心内容

### 短轮询（Short Polling）

客户端以固定间隔反复发送 HTTP 请求检查新数据。

- **优点**：实现极简，纯标准 HTTP
- **缺点**：网络开销大（大量空请求）、实时性低（延迟 = 0 到轮询间隔）、服务器负载高
- **适用**：简单应用、实时性要求不高的场景

### SSE（Server-Sent Events）

基于 HTTP 的标准，允许服务器通过持久化连接主动向客户端推送数据。**单向数据流**（Server → Client）。

| 特性 | 说明 |
|------|------|
| 协议 | 基于标准 HTTP（无需升级） |
| 方向 | 仅服务器→客户端 |
| 自动重连 | 浏览器内置 |
| 事件恢复 | 通过 Last-Event-ID 断点续传 |
| 数据格式 | 仅 UTF-8 文本 |
| 浏览器支持 | 97.23%（2026） |

#### SSE 并发连接数限制（重要）

**HTTP/1.1**：浏览器限制同一域名最多 **6 个并发连接**（跨所有标签页共享）。这个 6 连接限制并非来自 RFC 2616 的直接建议——RFC 2616 实际建议的是 2 个并发连接，6 个是浏览器厂商自行实现的惯例。Chrome 和 Firefox 均标记为 "Won't Fix"。

**HTTP/2**：通过多路复用消除 6 连接限制，默认支持约 **100 个并发流**。SSE 仅占用一个复用流。**生产环境 SSE 应使用 HTTP/2**。

**规避方案**：使用 HTTP/2 / 多子域名分散 / SharedWorker 共享连接 / Web Locks + BroadcastChannel

**Wire 格式**：
```
event: message\n
id: 123\n
data: {"type": "notification"}\n
\n
```

**典型场景**：AI 流式输出（ChatGPT、Claude）、实时通知、活动流、实时仪表盘

### WebSocket

RFC 6455 定义的全双工通信协议，在单个 TCP 连接上实现双向实时通信。

| 维度 | HTTP | WebSocket |
|------|------|-----------|
| 通信模式 | 请求-响应 | 全双工 |
| 数据方向 | 客户端发起 | 双向任意时刻 |
| 帧头开销 | 每次数百字节 | 2-14 字节 |
| 建立方式 | 直接连接 | HTTP Upgrade 握手 |

#### WebSocket 握手（101 协议切换）

连接始于 HTTP/1.1 升级请求：

```
客户端：GET /chat HTTP/1.1
        Upgrade: websocket
        Connection: Upgrade
        Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
        Sec-WebSocket-Version: 13

服务器：HTTP/1.1 101 Switching Protocols
        Upgrade: websocket
        Connection: Upgrade
        Sec-WebSocket-Accept: s3pPLMBiTxaQ9kYGzzhZRbK+xOo=
```

**关键细节**：
- 必须是 HTTP/1.1 的 GET 请求
- Sec-WebSocket-Key 不是安全机制，仅防止缓存代理误响应
- Sec-WebSocket-Accept = SHA-1(Key + 固定GUID) 的 Base64 编码
- 版本 13 是唯一广泛使用的版本
- 101 之外的任何状态码意味着握手失败
- wss://（TLS 加密）比 ws:// 更可靠，因为 TLS 隧道可绕过代理检查

**典型场景**：在线聊天、协同编辑、多人游戏、实时交易、IoT 控制

### 技术选型指南

```
需要实时通信？
├── 只需服务器推送？
│   └── SSE（简单高效，HTTP 原生，自动重连）
├── 需要双向通信？
│   ├── 文本为主 → WebSocket
│   ├── 二进制为主 → WebSocket
│   └── 企业代理环境 → WebSocket (wss://)
└── 实时性要求不高？
    └── 短轮询
```

| 场景 | 推荐技术 |
|------|---------|
| AI 流式输出 | SSE |
| 在线聊天 | WebSocket |
| 实时通知 | SSE |
| 股票行情 | WebSocket |
| 协同编辑 | WebSocket |
| 多人游戏 | WebSocket |
| 简单状态轮询 | 短轮询 |

**选 SSE 当**：只需推送、想要自动重连、需要穿越代理/CDN、HTTP/2 环境

**选 WebSocket 当**：需双向通信、极低延迟、二进制数据、高频消息
