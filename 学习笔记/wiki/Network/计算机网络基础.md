# 计算机网络基础

## 概述

计算机网络基础知识涵盖 OSI 模型、IP/MAC 地址、DNS 域名解析、子网掩码、网关等核心概念。这些是理解 HTTP 通信和网络编程的前置基础。

---

## OSI 七层模型

| 层级 | 名称 | 作用 | 代表协议 |
|------|------|------|----------|
| 7 | 应用层 | 给用户提供服务 | HTTP、HTTPS、FTP、DNS |
| 6 | 表示层 | 数据格式转换、加密 | SSL/TLS |
| 5 | 会话层 | 建立/管理会话 | RPC |
| 4 | 传输层 | 端到端通信 | TCP、UDP |
| 3 | 网络层 | 路由、IP 寻址 | IP、ICMP |
| 2 | 数据链路层 | MAC 寻址 | Ethernet、ARP |
| 1 | 物理层 | 比特流传输 | 网线、光纤 |

> **常见协议归属**：DNS 和 HTTP 工作在**应用层**，TCP 工作在**传输层**。

## MAC 地址 vs IP 地址

### MAC 地址

- 网卡的**物理地址**（硬件地址），通常固定不变
- 工作在**数据链路层**（OSI 第二层）
- 用途：**局域网通信**——交换机靠 MAC 转发数据，网卡靠 MAC 接收数据

### IP 地址

- **逻辑地址**，可变化
- 工作在**网络层**（OSI 第三层）
- 用途：**跨网络通信**

### 对比

| 维度 | MAC | IP |
|------|-----|-----|
| 类别 | 物理地址 | 逻辑地址 |
| 是否可变 | 通常固定 | 可变化 |
| 工作层 | 数据链路层 | 网络层 |
| 用途 | 局域网通信 | 跨网络通信 |

> **简单类比**：MAC 地址是你的身份证号（固定），IP 地址是你当前的住址（可以搬）。

## DNS 域名解析

DNS（Domain Name System）是域名解析系统，将域名转换为 IP 地址。例如 `baidu.com` → `110.xx.xx.xx`。

### 解析过程

```
客户端 → 本地DNS → 根DNS → 顶级域DNS → 权威DNS → 最终IP
```

1. 先搜索**本地缓存**，若无则向本地 DNS 求助
2. 本地 DNS 代表你去互联网询问
3. 本地 DNS 问**根域名服务器**，根服务器不知道具体 IP，但知道 `.com` 顶级域归谁管（返回 TLD 服务器地址）
4. 本地 DNS 问**顶级域 DNS**（TLD），TLD 查到有记录，返回权威 DNS 地址
5. 本地 DNS 问**权威域名服务器**，查找后回复最终 IP
6. 获得最终 IP，解析完成

### DNS 所在层级

DNS 协议工作在**应用层**。

## 子网掩码与地址结构

### IP 地址结构

IP 地址由两部分组成：
- **网络号**：标识设备属于哪个网络（如 `192.168.1`）
- **主机号**：标识网络中的哪一台机器（如 `.10`）

### 子网掩码

用于区分网络号和主机号。

**为什么要区分？** 路由器只关心**网络号**——数据到达目标网络后，再通过主机号找到具体设备。

### 特殊地址

| 地址 | 含义 |
|------|------|
| `0.0.0.0` | 监听地址，监听所有网卡（程序"等待别人连接"的地址） |
| `127.0.0.1` | 回环地址，只允许本机访问 |
| `localhost` | 主机名，映射到 `127.0.0.1` |
| `192.168.1.255` | 广播地址，向网段中所有设备发送数据 |

> **区别**：`0.0.0.0` 是"接受所有网卡的连接"，`127.0.0.1` 是"只接受本机的连接"。

## 网关

网关是离开当前网络的**出口**——不同网络之间的"大门"。

### 举例

```
你的电脑：192.168.1.10
路由器：  192.168.1.1  ← 这就是默认网关
```

当访问百度服务器时，电脑发现"目标不在本局域网"，于是把数据交给网关（路由器）。

## 常见请求头速查

| 请求头 | 作用 | 常见取值 |
|--------|------|----------|
| Host | 请求的域名 | `www.baidu.com` |
| User-Agent | 客户端信息 | Chrome/Firefox |
| Content-Type | 请求体类型 | `application/json` |
| Authorization | 身份认证 | `Bearer token` |
| Accept | 接收的数据类型 | `application/json` |
| Cookie | 携带 Cookie | `sessionid=xxx` |
| Referer | 来源页面 | `https://google.com` |
| Origin | 请求来源 | `https://xxx.com` |
| Connection | 是否保持连接 | `keep-alive` |
| Accept-Encoding | 压缩方式 | `gzip` |
| Accept-Language | 语言 | `zh-CN` |
| Content-Length | 请求体长度 | `348` |

### 常见 Content-Type

| 类型 | 含义 |
|------|------|
| `application/json` | JSON 数据 |
| `application/x-www-form-urlencoded` | 表单 |
| `multipart/form-data` | 文件上传 |
| `text/plain` | 纯文本 |
| `text/html` | HTML |
