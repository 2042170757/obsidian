# 大模型训练流程

## 概述

大模型的完整训练过程分为四个阶段：预训练 → 有监督微调（SFT）→ 人类反馈强化学习（RLHF）→ 低成本微调（LoRA）。每个阶段解决不同的问题，逐步将一个"什么都懂但不会聊天"的基座模型，打磨成"既聪明又贴心"的专用助手。

> **一句话理解**：预训练是义务教育（学知识），SFT 是职业培训（学技能），RLHF 是情商训练（懂人心），LoRA 是岗前进修（学专长）。

---

## 第一阶段：预训练（Pre-training）

**目标**：训练出一个通用的、了解所有知识的**基座模型**（Base Model）。

### 关键特点

| 维度 | 说明 |
|------|------|
| 数据要求 | **海量无标注**数据，万亿级 Token 规模，全类型知识 |
| 算力需求 | 超大规模算力（数千张 GPU） |
| 技术门槛 | 极高 |
| 模型能力 | 什么都懂，但不会听指令 |

### 预训练的本质

相当于对模型进行"义务教育"——模型是一个通才，什么都知道一点，但不会按照你的指令办事。预训练**决定模型的上限**。

---

## 第二阶段：有监督微调（SFT, Supervised Fine-Tuning）

**目标**：让模型听懂人话，具备**指令跟随能力**。

### 关键特点

| 维度 | 说明 |
|------|------|
| 数据要求 | 高质量的**指令 + 响应**键值对 |
| 数据量级 | 相比预训练大幅减少 |
| 核心能力 | 意图识别、指令遵循 |

### 训练数据格式

```
{"instruction": "请用一句话解释量子力学", "response": "量子力学是研究微观粒子行为的物理学分支..."}
```

### SFT 的作用

让模型从"什么都知道但不会聊天"变成"既知道又会按照你的要求回答"。

---

## 第三阶段：人类反馈强化学习（RLHF）

**目标**：让模型**懂人心**——更贴心、安全、符合伦理。

### 核心定位

- 对齐人类偏好
- 解决"不好用"的痛点
- 提升主观质量，对齐人类价值观

### 三个子阶段

```
SFT 监督微调 → RM 奖励模型训练 → RL 强化学习优化
```

1. **SFT 监督微调**：基础的指令跟随能力
2. **RM 奖励模型训练**：学习人类的偏好排序（哪个回答更好）
3. **RL 强化学习优化**：根据奖励信号调整模型行为

### 适用场景

- 主观创意类任务（写作、对话）
- 安全和价值观对齐的任务

---

## 第四阶段：低成本微调（LoRA）

**目标**：用极低的成本定制**领域专用模型**。

### 核心原理

- **冻结**基座模型的所有原始参数，不对其进行改动
- 引入**低秩矩阵**，只训练新增的少量参数
- 保留原始基座模型的**通用能力**

### 关键优势

| 维度 | 说明 |
|------|------|
| 硬件成本 | 大幅减少（消费级 GPU 可训练） |
| 训练速度 | 显著提升 |
| 模型能力 | 无损原模型通用能力 |
| 数据需求 | 小数据量场景即可 |

### 适用场景

- 小数据量的领域定制
- 有限算力环境
- 需要保留模型原始能力的场景

> **注意**：RAG 不需要训练模型；LoRA 固定原有基座模型，只修改新增矩阵。两者解决不同层面的问题。

## 模型名称后缀速查

常见模型后缀的含义：

| 后缀 | 含义 |
|------|------|
| VL | 视觉语言，多模态模型（可识别图片） |
| Code | 对编码环境优化过 |
| Instruct / it | 为指令优化过的模型 |
| Chat | 对话模型 |
| RL | 强化学习过的模型 |
| Lite | 轻量化（中杯） |
| Pro | 增强版本（正常杯） |
| Max | 加大杯 |
| Thinking | 推理模型 |
| Flash | 快速模型 |
| Math | 数学计算优化 |
| Base | 基座模型 |
| xxK | 上下文长度为 xxK Token |
| TTS | 语音合成模型 |

## 开源 vs 闭源模型

### 开源模型

模型权重、模型文件、代码公开。代表：DeepSeek、Qwen、LLaMA、GLM、Kimi。

优势：高度定制、低成本、技术透明、可微调、可蒸馏、可本地部署。

### 闭源模型

只对外提供模型服务，黑盒。代表：Claude、Gemini、GPT。

优势：一般来说模型能力较强。
