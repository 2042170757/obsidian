# 容器数据持久化

## 概述
容器的可写层在容器删除后会丢失。Docker 提供三种存储挂载方式来持久化数据：Volume（数据卷）、Bind Mount（绑定挂载）和 tmpfs（内存挂载）。

---

## 一、为什么需要数据持久化？

```
容器删除前                    容器删除后
┌──────────────────┐         ┌──────────────────┐
│ Container Layer  │         │                  │
│ (可写层：数据存在) │  ──►   │    (已删除)       │  ← 数据丢失！
├──────────────────┤         └──────────────────┘
│   Image Layers   │
│   (只读)          │
└──────────────────┘
```

**解决方案：** 将数据存储在容器外部，独立于容器生命周期。

---

## 二、三种存储方式对比

| 特性 | Volume（数据卷） | Bind Mount（绑定挂载） | tmpfs（内存挂载） |
|------|------------------|----------------------|-------------------|
| **存储位置** | Docker 管理的目录 (`/var/lib/docker/volumes`) | 宿主机任意路径 | 宿主机内存 |
| **持久性** | 是（直到手动删除） | 是（数据在宿主机上） | 否（容器停止即丢失） |
| **Docker 管理** | 是 | 否 | 否 |
| **跨平台** | 是 | 依赖宿主机路径 | 仅 Linux |
| **安全性** | 高（Docker 管理） | 中（容器可修改宿主机文件） | 高（无磁盘访问） |
| **主要用途** | 生产环境：数据库、日志、用户数据 | 开发环境：源码、实时热重载 | 临时数据：缓存、敏感信息 |

---

## 三、Volume（数据卷）

Volume 是 Docker **推荐的持久化方式**，完全由 Docker 管理。

### 核心命令

```bash
# 创建卷
docker volume create mydata

# 列出所有卷
docker volume ls

# 查看卷详情
docker volume inspect mydata

# 删除卷
docker volume rm mydata

# 删除所有未使用的卷
docker volume prune
```

### 使用方式

```bash
# 方式 1：命名卷（推荐）
docker run -d -v mydata:/app/data nginx

# 方式 2：匿名卷
docker run -d -v /app/data nginx

# 方式 3：只读挂载
docker run -d -v mydata:/app/data:ro nginx

# 方式 4：使用 --mount（更明确的语法）
docker run -d --mount type=volume,source=mydata,target=/app/data nginx
```

### Volume 的优势
- Docker 自动管理存储位置
- 可以在多个容器间共享
- 卷驱动支持远程存储（NFS、云存储等）
- 容器删除后数据仍然保留

## 四、Bind Mount（绑定挂载）

Bind Mount 将宿主机上的文件或目录直接映射到容器中。

### 使用方式

```bash
# 基本语法
docker run -d -v /宿主机路径:/容器路径 nginx

# 示例：挂载当前目录
docker run -d -v $(pwd):/app nginx

# 只读挂载
docker run -d -v $(pwd)/config:/etc/nginx/conf.d:ro nginx

# 使用 --mount 语法
docker run -d --mount type=bind,source=$(pwd),target=/app nginx
```

### 开发热重载场景

```bash
# 源代码在宿主机上修改，容器内实时生效
docker run -d -v $(pwd)/src:/app/src -p 3000:3000 node:alpine npm run dev
```

### 注意事项
- 如果宿主机路径不存在，`-v` 会自动创建为目录，`--mount` 会报错
- 挂载会**遮盖**容器内该路径的原有内容
- 宿主机和容器可以同时修改挂载的文件

---

## 五、tmpfs（内存挂载）

tmpfs 将数据存储在**宿主机内存**中，不写入磁盘。

### 使用方式

```bash
# 基本用法
docker run -d --tmpfs /app/temp nginx

# 使用 --mount 指定大小和权限
docker run -d --mount type=tmpfs,target=/app/temp,tmpfs-size=104857600,tmpfs-mode=1777 nginx
```

### 适用场景
| 场景 | 说明 |
|------|------|
| 敏感数据 | 密码、令牌等不应写入磁盘的数据 |
| 临时缓存 | 中间计算结果、临时文件 |
| 高性能 I/O | 避免磁盘 I/O 瓶颈 |

### 限制
- 仅 Linux 支持
- 不能在容器间共享
- 容器停止后数据丢失
- 默认最大容量为宿主机 RAM 的 50%

## 六、Docker Compose 中的存储配置

```yaml
services:
  db:
    image: postgres:16
    volumes:
      - postgres_data:/var/lib/postgresql/data  # 命名卷
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql  # 绑定挂载
    environment:
      POSTGRES_PASSWORD: secret

  app:
    image: myapp:latest
    volumes:
      - ./src:/app/src  # 绑定挂载（开发用）
      - app_temp:/tmp   # 命名卷

volumes:
  postgres_data:  # 声明命名卷
  app_temp:       # 声明命名卷
```
