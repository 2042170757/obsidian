# 容器生命周期与管理

## 概述
理解容器从创建到销毁的完整生命周期，以及日常管理容器的核心命令。

---

## 一、容器生命周期

### 状态流转图

```
                    docker create
                         │
                         ▼
                    ┌─────────┐
                    │ created │
                    └────┬────┘
                         │
                    docker start
                         │
                         ▼
    docker unpause  ┌─────────┐   docker pause
         ┌──────────│ running │◄──────────┐
         │          └────┬────┘           │
         │               │                │
         │          ┌────▼────┐           │
         └─────────►│ paused  │───────────┘
                    └─────────┘
                         │
            docker stop / docker kill
                         │
                         ▼
                    ┌─────────┐
                    │ exited  │
                    └────┬────┘
                         │
                    docker rm
                         │
                         ▼
                    (已删除)
```

### 核心状态定义

| 状态 | 说明 | 触发命令 |
|------|------|----------|
| **created** | 容器已创建但未启动 | `docker create` |
| **running** | 容器正在运行 | `docker start` / `docker run` |
| **paused** | 容器进程被暂停（SIGSTOP） | `docker pause` |
| **exited** | 容器已停止（进程退出） | `docker stop` / `docker kill` / 进程自行退出 |
| **restarting** | 容器正在重启 | 重启策略触发 |
| **removing** | 容器正在被删除 | `docker rm` |
| **dead** | 容器异常状态 | 错误导致 |

> **常见误区：停止 ≠ 删除！**
> - `docker stop` 只是停止容器进程，容器对象（包括可写层数据）仍然存在
> - `docker rm` 才会真正删除容器对象
> - 停止的容器仍然占用磁盘空间（可写层）

## 二、docker run 核心参数

`docker run` = `docker create` + `docker start`，是最常用的命令。

### 核心参数速查表

| 参数 | 说明 | 示例 |
|------|------|------|
| `-d` | 后台运行（detach 模式） | `docker run -d nginx` |
| `-it` | 交互式 + 伪终端（`-i` + `-t` 组合） | `docker run -it ubuntu bash` |
| `--rm` | 容器退出后自动删除 | `docker run --rm nginx echo hello` |
| `-p` | 端口映射（HOST:CONTAINER） | `docker run -p 8080:80 nginx` |
| `-P` | 发布所有 EXPOSE 端口到随机端口 | `docker run -P nginx` |
| `-e` | 设置环境变量 | `docker run -e MYSQL_ROOT_PASSWORD=secret mysql` |
| `--name` | 给容器命名 | `docker run --name myweb nginx` |
| `-v` | 挂载卷 | `docker run -v data:/app/data nginx` |
| `--network` | 指定网络 | `docker run --network mynet nginx` |
| `--restart` | 重启策略 | `docker run --restart unless-stopped nginx` |
| `-w` | 设置工作目录 | `docker run -w /app nginx` |
| `--entrypoint` | 覆盖入口点 | `docker run --entrypoint /bin/sh nginx` |

### 端口映射详解

```bash
# 格式：-p HOST_PORT:CONTAINER_PORT
docker run -d -p 8080:80 nginx
# 访问 http://localhost:8080 → 转发到容器的 80 端口

# 映射到随机端口
docker run -d -p 80 nginx
# 使用 docker port <容器ID> 查看实际映射

# 映射多个端口
docker run -d -p 8080:80 -p 8443:443 nginx

# 映射到指定 IP
docker run -d -p 127.0.0.1:8080:80 nginx

# -P：发布所有 EXPOSE 端口到随机端口
docker run -d -P nginx
```

### 常见服务端口

| 服务 | 默认端口 | 示例 |
|------|----------|------|
| HTTP/Web | 80, 8080 | `nginx`, `apache` |
| MySQL | 3306 | `mysql` |
| PostgreSQL | 5432 | `postgres` |
| Redis | 6379 | `redis` |
| SSH | 22 | `sshd` |
| MongoDB | 27017 | `mongo` |
| HTTPS | 443 | Web 服务 |

### 环境变量注入

```bash
# 单个变量
docker run -e MYSQL_ROOT_PASSWORD=secret mysql

# 多个变量
docker run -e DB_HOST=localhost -e DB_PORT=3306 myapp

# 从文件读取
docker run --env-file .env myapp
```

## 三、容器日常管理命令

### 查看容器

| 命令 | 说明 |
|------|------|
| `docker ps` | 查看正在运行的容器 |
| `docker ps -a` | 查看所有容器（包括已停止的） |
| `docker ps -q` | 只输出容器 ID |
| `docker ps -s` | 显示容器大小 |
| `docker inspect <容器>` | 查看容器详细信息（JSON 格式） |
| `docker stats` | 实时显示所有容器的资源使用情况 |
| `docker stats <容器>` | 查看指定容器的资源使用 |

### 停止和删除

| 命令 | 说明 | 信号 |
|------|------|------|
| `docker stop <容器>` | 优雅停止（发送 SIGTERM，超时后 SIGKILL） | SIGTERM → SIGKILL |
| `docker kill <容器>` | 强制停止（直接发送 SIGKILL） | SIGKILL |
| `docker rm <容器>` | 删除已停止的容器 | - |
| `docker rm -f <容器>` | 强制删除运行中的容器 | - |

### 清理

| 命令 | 说明 |
|------|------|
| `docker system prune` | 删除所有停止的容器、未使用的网络、悬空镜像 |
| `docker system prune -a` | 删除所有未使用的镜像（不仅是悬空的） |
| `docker system prune --volumes` | 同时删除未使用的卷 |
| `docker container prune` | 只删除停止的容器 |

### 重启策略

```bash
# no：不自动重启（默认）
docker run --restart no nginx

# on-failure：非正常退出时重启（可指定最大重试次数）
docker run --restart on-failure:5 nginx

# always：总是重启（除非手动停止）
docker run --restart always nginx

# unless-stopped：类似 always，但 Docker 守护进程重启后不自动启动
docker run --restart unless-stopped nginx
```

## 四、调试利器

### docker exec — 进入运行中的容器

```bash
# 交互式进入容器（最常用）
docker exec -it <容器名或ID> /bin/bash

# 如果容器没有 bash（如 Alpine）
docker exec -it <容器名或ID> /bin/sh

# 在容器中执行单条命令
docker exec <容器名或ID> cat /etc/nginx/nginx.conf

# 以指定用户执行
docker exec -u root -it <容器名或ID> /bin/bash

# 设置环境变量
docker exec -e MY_VAR=hello <容器名或ID> env
```

### docker logs — 查看容器日志

```bash
# 查看所有日志
docker logs <容器名或ID>

# 实时跟踪日志（类似 tail -f）
docker logs -f <容器名或ID>

# 显示最后 100 行
docker logs --tail 100 <容器名或ID>

# 显示时间戳
docker logs -t <容器名或ID>

# 查看指定时间之后的日志
docker logs --since 2024-01-01T00:00:00 <容器名或ID>

# 查看指定时间之前的日志
docker logs --until 2024-01-02T00:00:00 <容器名或ID>
```
