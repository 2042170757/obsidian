# AGENTS.md - 学习笔记知识库 Schema

基于 [Karpathy LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) 模式构建的个人知识库。

---

## 核心理念

**你（AI）负责维护这个 wiki，用户很少（或不）直接写 wiki。**

- 用户负责：提供原始来源、提问、探索
- 你负责：总结、交叉引用、归档、维护

> 想象：Obsidian 是 IDE，你 是程序员，wiki 是代码库。

---

## 架构

```
学习笔记/
├── .raw/                    # 原始来源（append-only. Never edit）
│   ├── articles/           # 文章
│   ├── papers/             # 论文
│   └── clips/              # 剪藏
│
├── wiki/                   # LLM 生成的 wiki（由 AI 维护）
│   ├── index.md            # 内容目录（按分类）
│   ├── log.md              # 时间记录（按时间线）
│   ├── AI/                 # AI 相关
│   │   └── karpathy-llm-wiki.md
│   ├── Docker/             # Docker 相关
│   └── 文学/               # 文学相关
│
└── AGENTS.md               # 本文件 - Schema
```

## Wiki 页面类型

每个概念/主题一个独立的 .md 文件：

| 类型 | 说明 | 示例 |
|------|------|------|
| **概念页** | 定义和解释一个概念 | 什么是 Transformer |
| **实体页** | 描述一个实体（人物、作品） | Karpathy、红楼梦 |
| **摘要页** | 对来源的总结 | Attention 论文摘要 |
| **比较页** | 对比两个或多个概念 | RNN vs Transformer |
| **综合页** | 汇总多个来源的观点 | AI 辅助编程 |

## 工作流程

### 1. Ingest（摄入新文档）

**步骤**：

```
1. 先读取 index.md，了解现有 wiki 结构
2. 阅读原始文档，提取关键概念和信息
3. 判断：
   - 新概念 → 创建新页面
   - 已存在 → 询问用户还是更新
4. 创建/更新 wiki 页面
   - 每个概念一个文件
   - 使用 [[wiki/分类/页面名]] 链接
5. 更新 index.md（添加新页面链接）
6. 更新 log.md（记录本次摄入）
7. 检查矛盾：
   - 如果新信息与旧内容矛盾 → 明确标记
```

### 2. Query（回答问题）

**优先级**：

```
1. 首先在 wiki 中查找相关信息（优先）
2. 如果 wiki 没有完整答案，再查看 .raw/ 原始来源
3. 综合回答，并引用相关 wiki 页面
4. 如果发现新的可整合信息，主动建议更新 wiki
```

### 3. Lint（健康检查）

定期（或按需）检查 wiki 健康状态：

```
1. 检查孤立页面（没有被任何页面链接）→ 添加链接或删除
2. 检查断链（指向不存在的页面）→ 修复或创建
3. 检查过时信息（与最新来源矛盾）→ 标记矛盾
4. 报告问题并建议修复
```

## 页面格式规范

### 命名规则
- 使用小写字母
- 用连字符分隔：`machine-learning.md`
- 中文用中文：`机器学习.md`

### 必需结构

```markdown
# 页面标题

## 概述
（一句话说明这是什么）

## 核心内容
（详细笔记）

## 相关概念
- [[相关页面1]]
- [[相关页面2]]

## 来源
- [[.raw/clips/xxx.md]]
```

### 交叉引用规范

- 同一分类内：用 `[[wiki/AI/页面名]]`
- 跨分类：用 `[[wiki/分类/页面名]]`
- **每个新页面至少链接到 1 个已有页面**，避免孤立页面

### 矛盾处理

当新来源与旧内容矛盾时：

```markdown
## ⚠️ 矛盾记录

- 来源A 说 X
- 来源B 说 Y（与此矛盾）
- 待验证
```

## 触发词

- **"摄入" / "添加"** → 执行 Ingest 流程
- **"查询" / "搜索"** → 执行 Query 流程
- **"检查" / "lint"** → 执行 Lint 流程
- **"新建分类"** → 创建新的分类文件夹
