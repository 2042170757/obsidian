# 三，大模型核心参数
11. **token是大模型理解文本的最小语义单位**，token是模型处理的单位（32k），**上下文窗口的单位，计费单位**

12. **理解常见模型的分词，不同模型分词不同，计算出来的token数量也就不同，理解token和汉字，词语，英文字母，单词的换算单位，比如：**

> [!tip] 经验换算
> - 英文：1 token ≈ 0.75 单词 ( 4 字母 )
> - 中文：1 token ≈ 0.6~1.2 汉字
> - 1000 tokens ≈ 750 单词 ≈ 600-1200 汉字
> - 分词器差异！！



什么是上下文窗口？
**最大**模型处理任务，决定**记忆容量**，**输入**+**输出**的上限

上下文窗口的工作机制：
**动态滑动窗口**，聚焦**窗口内**token，稀疏计算

13. 上下文窗口：动态的，滑动窗口，超出窗口的部分，模型会忽略，输入token + 输出token <= 上下文窗口

超出上下文窗口会怎么样？
超出的部分会被**舍弃掉**，模型**无法参考**超出窗口的内容，长文本处理信息丢失**总结失真**

长文本总结的坑：
根源分析：**上下文超限**；解决方案：**分块+递归总结**

14. 超出上下文窗口的现象：信息丢失，模型记不住，无法回答超出部分的内容，输出不完整，理解max_tokens参数
15. 常见的上下文窗口大小：4k，32k，128k，1M
16. 超出上下文窗口的解决方法：分块，使用更大窗口的模型，精简提示词，开启新对话/new，压缩上下文/compact

*大模型训练全过程*：
1.**预训练** （基座模型）
2.**有监督微调** （具备指令跟随能力的对话式模型）
3.**人类反馈强化学习** （更贴心，安全，且符合伦理的智能模型）
4.**低成本微调**（低成本，高性能的领域专用定制模型）

17. 模型训练：预训练->SFT->RLHF，LoRA

预训练：大模型的义务教育和底层基建
数据要求：**海量无标注**数据，万亿级token规模，全类型知识
适配需求：海量数据，超大规模算力，极高技术门槛

18. 预训练：训练出一个通用的，了解所有知识的基座模型，相当于对模型进行义务教育，模型是一个通才，什么都懂，对算力，数据量，算法要求很高，决定模型的上限

监督微调：SFT
让模型听懂你的话：意图识别
数据要求：高质量指令，低量级需求，真实交互场景

19. SFT：有监督微调，让模型理解并遵循指令，训练数据为：指令+响应的键值对

RLHF：让模型懂人心
核心定位：对齐人类偏好
核心价值：解决不好用的痛点
三个阶段：**SFT监督微调，RM奖励模型训练，RL强化学习优化**
核心作用：***提升主观质量，对齐人类价值观，贴合使用习惯***

20. RLHF：基于人类反馈的强化学习，对齐人类普世价值观，训练三个阶段：SFT->RM奖惩训练->RL强化学习，适用场景：主观创意类任务,安全和价值观对齐的任务

LoRA：微调神器
核心定位：参数高效微调
核心原理：冻结模型参数，引入低秩矩阵，高效增量训练
核心优势：*硬件成本减少，训练速度显著提升，无损原模型*
适配需求：*小数据量场景，有限算力适配，保留通用能力*

21. LoRA：低成本，低参数，低数据量的微调技术，固定基座模型的参数，不对其进行改动，只修改新增的矩阵，保留了原始基座模型的通用能力，提升训练速度，降低训练时间，成本，小样本场景，有限算力场景，保留模型原始能力的场景

49. 量化：不改变模型参数量，对模型的参数精度做调整，原始精度是FP32，4字节，常见量化int4，int8，FP16
50. 简单判断显存用量：4位量化下，模型参数量如7B，就表示需要使用大约7G显存，
51. 显存使用量计算公式：模型参数量 x 量化精度的字节数 x 1.2 比如int4，4位量化单参数位0.5字节，7B模型的显存用量位：7 x 0.5 x 1.2 
52. 常见量化后缀：Q4_K_M、INT4、INT8、GGUF、GPTQ、AWQ

58. 了解dns的工作原理，能讲出来当在浏览器中打开一个网址的全过程
首先打开一个网站，设备先查询**本地缓存**，**hosts**如果没有，则向**本地DNS**发起请求
本地DNS先向**根DNS**查询，根DNS返回顶级域DNS
本地DNS向**顶级域DNS**（TLD）查询，顶级域DNS返回权威DNS
本地DNS向**权威DNS**查询，返回IP

拿到IP地址了，要开始建立TCP连接了，三次握手：
第一次客户端：SYN，
第二次：服务器：SYN+ACK,
第三次：客户端：ACK

广播地址：向一个网段中的所有设备发送数据；

GET和POST区别：GET是获取数据，参数在URL上，长度有限制，可缓存；
POST是提交数据，参数在body上，长度较大，一般不缓存

交换机的作用：交换机看MAC，路由器看IP；

什么是MAC地址？网卡的物理地址（硬件地址）
MAC地址的作用：局域网通信的时候，交换机靠MAC转发数据，网卡靠MAC接收数据
MAC地址&工作在哪一层：MAC工作在数据链路层（OSI第二层），IP工作在网络层（OSI第三层）
MAC负责局域网定位，IP负责跨网络定位；


| 对比   | MAC   | IP    |
| ---- | ----- | ----- |
| 类别   | 物理地址  | 逻辑地址  |
| 是否可变 | 通常固定  | 可变化   |
| 工作层  | 数据链路层 | 网络层   |
| 用途   | 局域网通信 | 跨网络通信 |
