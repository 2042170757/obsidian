1. 了解127.0.0.1和localhost，使用虚拟机的nginx起一个web服务，在虚拟机中访问这个服务，再在win上访问这个服务


**127.0.0.1** 是一个特殊的 IP 地址，叫做回环地址。它永远指向"本机自己"，数据包发出去后不会经过任何网卡，直接在操作系统内核里转一圈就回来了。(直接)
**localhost** 是一个主机名，在几乎所有操作系统的 `hosts` 文件中，它都被映射到 `127.0.0.1`。所以访问 `localhost` 等价于访问 `127.0.0.1`。

> 严格来说，`localhost` 可以被修改映射到其他地址，但实际中几乎不会这样做。
**地址范围**：整个 `127.0.0.0/8` 段（即 `127.0.0.1` ~ `127.255.255.254`）都保留作为回环地址，但最常用的就是 `127.0.0.1`。

- **127.0.0.1** 就像是你**自己家的门牌号**。你写信寄给"自己家"，邮局发现地址就是你自己，信根本不用出门，在家里转一圈就回到了你手上。这个过程不经过外面的马路（网卡/网络），速度极快。
- **localhost** 就像是"**我家**"这个口语说法。它和写具体门牌号（127.0.0.1）效果一样，只是叫法不同。`localhost` → `127.0.0.1` 的过程，就像别人说"去你家"，你脑子里自动翻译成具体的门牌地址一样。

3. 配置openclaw *QQ* 渠道

先去官网 *QQ开放平台* ，注册/登录
点击*创建机器人*
创建之后保存 *AppID和AppSecret* 


- 第一先安装插件 openclaw plugins install @openclaw/qqbot

![PixPin_2026-05-06_21-55-58.jpg](https://gitee.com/linzhipeng123456/images/raw/main/20260506215620269.jpg)


- 第二设置openclaw.json的文件
添加qq.bot
```"channels": {
    "qqbot": {
      "enabled": true,
      "appId": "1903461872",
      "clientSecret": "AppID(我直接隐藏一下)",
      "allowFrom": ["AppSecret（我再隐藏一下）"],
      "streaming": true
    }
  },
```

- 第三插件注册时添加的部分
```"plugins": {
    "entries": {
      "deepseek": {
        "enabled": true
      },
      "qqbot": {
        "enabled": true
      }
    }
  },
```

- 第四部分重启网关
```
openclaw gateway restart
```
